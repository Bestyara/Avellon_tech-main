### Цель

Сделать так, чтобы при нажатии кнопки **«Открыть проект»** в главном меню открывалось модальное окно со списком проектов из БД, откуда пользователь выбирает один проект и открывает его (двойным кликом или кнопкой «Открыть»).

### Ключевые места в коде

- **Главное окно и главное меню**: `[main_window.py](main_window.py)`
  - Класс `MainWindow` (методы `run_main_menu`, `open_project_by_path`).
  - Класс `MainMenuWidget` (метод `open_project_action`, сейчас открывает диалог выбора папки `select_path_to_dir`).
- **Работа с БД**: `[db_storage.py](db_storage.py)`
  - Класс `DbStorage` (есть методы `create_project`, `get_project_by_path`, `get_project_by_id`, но нет получения списка всех проектов).

### План изменений

1. **Расширить API хранилища проектов в `DbStorage`**
  - Добавить метод, возвращающий список всех проектов, отсортированных по дате создания или имени, например `list_projects() -> List[Dict[str, Any]]`.
  - Формат элементов списка: как минимум `project_id`, `project_name`, `project_path`, `created_at`.
2. **Создать диалог выбора проекта**
  - В `main_window.py` добавить новый класс, например `OpenProjectDialog(QDialog)` или на основе существующего абстрактного диалога (`AbstractToolDialog`, если он удобен для единообразия).
  - В диалоге:
    - В конструкторе принимать ссылку на `MainWindow` или `DbStorage`.
    - При инициализации запрашивать у `DbStorage` список проектов (`list_projects`).
    - Отобразить проекты в `QTableWidget` или `QListWidget`, где каждая строка представляет один проект.
      - Колонки/поля: **только имя проекта** (как вы выбрали). При необходимости сохранить `project_id`/`project_path` в `QTableWidgetItem.setData()`.
    - Добавить кнопку **«Открыть»** и кнопку **«Отмена»** снизу.
    - Поведение выбора:
      - **Двойной клик по строке** — сразу закрывает диалог с `accept()` и запоминает выбранный проект.
      - Кнопка **«Открыть»** — активна, когда выбрана строка; по нажатию тоже вызывает `accept()`.
    - Предусмотреть случай, когда в БД нет ни одного проекта: показывать пустой список и/или текст «Нет доступных проектов», кнопка «Открыть» тогда неактивна.
3. **Интеграция диалога с `MainMenuWidget`**
  - В `MainMenuWidget.open_project_action` вместо вызова `select_path_to_dir` создать и показать `OpenProjectDialog`.
  - После `exec()` диалога:
    - Если диалог завершён с `Accepted`, получить выбранный `project_id` (или `project_path`) из диалога.
    - Через `MainWindow.db.get_project_by_id` (или напрямую из данных диалога) получить `project_path`.
    - Вызвать `self.main_window.run_borehole_menu(project_path, project_id=project_id)`.
  - При `Rejected` (или если пользователь ничего не выбрал) — просто выйти из метода без действий.
4. **Обработка ошибок и валидация**
  - Если выбранный проект есть в БД, но его папка `project_path` больше не существует на диске:
    - Показать `MessageBox().warning` с понятным текстом (по аналогии с уже реализованными предупреждениями в `MainWindow.open_project_by_path` и `open_last_project`).
    - Не падать, просто не открывать проект.
  - При получении списка проектов из БД игнорировать записи с пустым `project_path` или обрабатывать их отдельно (в простом варианте можно просто их не показывать в списке).
5. **Поддержка текущей логики кэша последнего проекта**
  - Не менять существующую логику `__cache_init`, `__cache_save_project_id`, `open_last_project` — они должны продолжить работать как есть.
  - При открытии проекта из нового диалога обязательно вызывать `run_borehole_menu` с `project_id`, чтобы кэш `save_last_project_id` продолжал обновляться.
6. **Минимальная проверка поведения (мануально)**
  - Запустить приложение.
  - Создать новый проект через «Создать проект» (убедиться, что он попадает в БД).
  - Вернуться в главное меню.
  - Нажать «Открыть проект»:
    - Убедиться, что открывается новое модальное окно со списком проектов.
    - Проверить открытие через двойной клик и через выбор строки + кнопку «Открыть».
  - Убедиться, что кнопка «Открыть последний проект» по-прежнему работает с выбранным ранее проектом.

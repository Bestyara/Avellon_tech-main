Проект нацелен на работу с данными, полученными со скважины

На данный момент функционал следующий:
- Создание проекта, как отдельной скважины
- Добавление секций в скважину, их настройка (имя, глубина, длина) и удаление. Добавление шагов измерения в каждую секцию. Добавление файлов в каждый шаг.
- Построение Осциллограммы по выбранным файлам.
- Построение Частотный характеристики для всех датчиков во всех шагах
- Построение Розы ветров по датчикам
- Построение Графика зависимости амплитуды во времени
- Построение Графика глубинной характеристики по шагам
- Конвертер файлов

Структура проекта:
- **./resource** - всякие картинки
- **./app.bat** - скрипт для запуска программы на Windows
- **./config.py** - файл с константными значениями в основном
- **./init.bat** - скрипт иницализации проекта на Windows
- **./Main.py** - корень проекта, файл, через который идет запуск проекта
- **./borehole_logic.py** - основная бизнес логика проекта
- **./third_party.py** - вспомогательные функции и классы
- **./README.md** - основная информация по проекту
- **./db_storage.py** - работа с БД - в процессе перехода к хранению информации в БД
- **./converter.py** - конвертация файлов - по-хорошему выпилить это при переходе к хранению инфы в БД
- **./loadlabel.py** - вроде функции для работы с многопоточкой, не уверен
- **./formatting.py** - функции для форматирования вывода информации
- **./main_window.py** - отрисовка окон тут происходит

Ключевые выводы по завершению работ с БД:
- БД является источником истины для структуры скважины: `projects` -> `boreholes` -> `sections` -> `steps`.
- Построение графиков работает в DB-first режиме: при наличии `file_id` (`db_file_id`) данные берутся из таблицы `files`, без зависимости от наличия исходного CSV на диске.
- Привязка файлов к шагам при загрузке из БД выполняется через `part_of_file_id` (фактически номер шага) и `file_id`.
- Связи для графиков (`frequency_characteristics`, `wind_roses`) сохраняются по `file_id`, а не через `file_path`.
- Файловый кэш последнего проекта и файловый fallback в основном DB-потоке удалены/отключены.
- Старые файловые методы синхронизации в UI переведены в no-op (DB-only режим), чтобы не было скрытой записи структуры в ФС.
